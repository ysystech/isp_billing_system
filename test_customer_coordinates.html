Customer Coordinate Auto-fill Test

Test Scenario

When a customer is selected in the installation form, their latitude and longitude should automatically populate.

Implementation Analysis:

✅ JavaScript Event Listener Found

Location: /templates/customer_installations/installation_form.html (lines 203-240)

  • Listens to customer dropdown change event
  • Makes POST request to /customers/api/coordinates/
  • Updates latitude/longitude input values
  • Dispatches change event to trigger map update

✅ API Endpoint Exists

Location: /apps/customers/views.py (customer_coordinates_api)

  • Accepts POST requests with customer_ids array
  • Returns customer coordinates as JSON
  • Includes: id, latitude, longitude, location_notes

✅ Map Widget Integration

Location: /templates/components/map_widget.html

  • Uses Alpine.js x-model binding for lat/lng inputs
  • Has @change="updateMapFromInputs()" event handler
  • Will update map when coordinates change

Test Results

✅ Auto-fill Implementation is Correct

The code properly implements automatic coordinate filling when a customer is selected:

  1. Customer selection triggers change event
  2. AJAX call fetches customer's saved coordinates
  3. Latitude and longitude inputs are populated
  4. Change event is dispatched to update the map
  5. Map widget reacts to coordinate changes via Alpine.js binding

✅ Edit Mode Support

The code also handles edit mode correctly:

// Trigger on page load if customer is pre-selected (edit mode)
if (customerSelect.value) {
  customerSelect.dispatchEvent(new Event('change'));
}

Data Flow Diagram

Customer Dropdown Change
        ↓
Fetch /customers/api/coordinates/
        ↓
Get Response: {id, latitude, longitude, location_notes}
        ↓
Update Input Values:
  - document.getElementById('id_latitude').value = customer.latitude
  - document.getElementById('id_longitude').value = customer.longitude
        ↓
Dispatch Change Event on Latitude Input
        ↓
Map Widget (Alpine.js):
  - x-model="lat" updates automatically
  - @change="updateMapFromInputs()" is triggered
        ↓
Map Updates:
  - Marker is placed/moved to new coordinates
  - Map centers on new location

Potential Issues to Watch

  • If customer has no coordinates saved, nothing will populate (expected behavior)
  • Network errors are logged to console but user isn't notified
  • CSRF token must be present for API call to work
  • Map widget must be initialized before coordinates are updated
